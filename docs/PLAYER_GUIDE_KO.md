# AdvancementBorder 플레이어 가이드

이 문서는 Minecraft Java Edition 26.2에서 AdvancementBorder를 처음 설치하고 플레이하는 사람을 위한 사용 설명서입니다.

AdvancementBorder는 월드를 매우 작은 월드보더에서 시작하게 하고, 플레이어가 발전과제를 완료할 때마다 오버월드·네더·엔드의 월드보더를 함께 확장합니다. 발전과제를 명령어로 취소하면 현재 진행 상태에 맞춰 보더도 다시 줄어듭니다.

## 1. 필요 환경

| 항목 | 요구 사항 |
|---|---|
| Minecraft | Java Edition 26.2 |
| 모드 로더 | Fabric Loader 0.19.3 이상 |
| 필수 모드 | Fabric API 0.154.2+26.2 또는 호환되는 26.2 버전 |
| Java | Java 25 |
| 플레이 방식 | 싱글플레이 기준 |

다른 Minecraft 버전용 모드 파일은 사용할 수 없습니다. 게임이 시작되지 않는다면 Minecraft, Fabric Loader, Fabric API, AdvancementBorder의 대상 버전이 모두 26.2인지 먼저 확인하세요.

## 2. 설치 방법

1. Minecraft 26.2용 Fabric Loader를 설치합니다.
2. Fabric API 26.2 버전을 준비합니다.
3. `advancementborder-1.0.0+26.2.jar`와 Fabric API JAR을 Minecraft의 `mods` 폴더에 넣습니다.
4. Fabric 프로필로 게임을 실행합니다.
5. 월드를 열면 `config/advancementborder.json`이 자동으로 생성됩니다.

기존 월드에서도 사용할 수 있지만, 보더가 줄어들면서 이동이나 탐험 범위가 크게 달라질 수 있으므로 월드 백업을 권장합니다.

## 3. 빠른 시작

### 3.1 새 월드 만들기

일반적인 싱글플레이 월드를 생성합니다. AdvancementBorder의 모든 명령은 일반 플레이어 권한으로 사용할 수 있으므로 치트를 켜거나 LAN 월드를 열 필요가 없습니다.

### 3.2 시작 블록 정하기

오버월드에서 최초 1×1 영역으로 사용할 블록 위에 섭니다.

```mcfunction
/advboard set ~ ~ ~
```

명령을 실행한 플레이어가 이 월드의 진행 소유자로 저장됩니다. 이후에는 그 플레이어의 발전과제만 보더 크기에 반영됩니다.

`~ ~ ~`는 현재 위치를 의미합니다. 특정 블록을 직접 지정하려면 다음처럼 입력할 수 있습니다.

```mcfunction
/advboard set 120 68 -35
```

월드보더 중심은 지정한 블록의 중앙인 `X+0.5`, `Z+0.5`에 배치됩니다. Y 좌표는 보더 크기에는 사용되지 않으며 안전 이동 기준 위치로 저장됩니다.

### 3.3 적용 상태 확인하기

```mcfunction
/advboard status
```

설정 직후 발전과제를 완료하지 않았다면 기본 보더 크기는 1블록입니다.

```text
[AdvancementBorder 상태]
활성 여부: 활성
완료 발전과제: 0개
초기 보더 크기: 1블록
발전과제당 증가량: 2블록
목표 보더 크기: 1블록
적용 보더 크기: 1블록
...
```

설정 전에 발전과제를 완료했다면 `/advboard set`을 실행하는 즉시 현재 완료 개수에 맞는 크기가 적용됩니다. 기존 발전과제는 초기화되지 않습니다.

## 4. 보더 성장 규칙

보더 크기는 다음 공식으로 계산됩니다.

```text
보더 한 변 길이 = 초기 보더 크기 + 완료 발전과제 수 × 발전과제당 증가량
```

기본 설정은 다음과 같습니다.

```text
초기 보더 크기: 1
발전과제당 증가량: 2
성장 순서: 1 → 3 → 5 → 7 → 9 ...
```

증가량을 1로 바꾸면 다음처럼 성장합니다.

```text
초기 보더 크기: 1
발전과제당 증가량: 1
성장 순서: 1 → 2 → 3 → 4 → 5 ...
```

짝수 크기가 되더라도 보더 중심은 이동하지 않습니다. 예를 들어 1에서 2로 증가하면 동·서·남·북 경계가 각각 0.5블록씩 확장됩니다. 이 때문에 짝수 크기에서는 보더 경계가 블록 중앙을 지날 수 있습니다.

## 5. 카운트되는 발전과제

다음 조건을 모두 만족하는 발전과제만 보더 크기에 반영됩니다.

- 발전과제 화면에 표시 가능한 발전과제
- 루트 노드가 아닌 발전과제
- 모든 완료 조건을 충족한 발전과제

다음 항목은 카운트되지 않습니다.

- `Minecraft`, `Nether`, `The End`, `Adventure`, `Husbandry` 등 각 탭의 시작 루트 과제
- 발전과제 화면에 표시되지 않는 레시피·내부 데이터 과제
- 조건을 일부만 달성한 미완료 발전과제

Minecraft 기본 발전과제뿐 아니라 다른 모드나 데이터팩이 추가한 표시 가능한 비루트 발전과제도 카운트됩니다. 모드팩 구성에 따라 최종적으로 얻을 수 있는 보더 크기가 달라질 수 있습니다.

일반 플레이로 발전과제를 달성하거나 `/advancement grant`, `/advancement revoke`로 상태를 변경하면 자동으로 다시 계산됩니다. 자동 재계산 시에는 채팅에 보더 크기 로그를 출력하지 않습니다. 현재 크기는 `/advboard status`로 확인하세요.

## 6. 차원별 보더 규칙

### 오버월드

- `/advboard set`으로 지정한 블록을 중심으로 사용합니다.
- 보더 크기 계산의 기준 차원입니다.

### 네더

- 오버월드와 동일한 X/Z 중심을 1:1로 사용합니다.
- 오버월드 보더가 7블록이면 네더 보더도 7블록입니다.
- 이 규칙은 월드보더 좌표에만 적용됩니다. 바닐라 포탈의 8:1 좌표 변환 규칙은 변경하지 않습니다.

### 엔드

- 기본 중심 블록은 X `100`, Z `0`입니다.
- 실제 보더 중심은 해당 블록 중앙인 `100.5`, `0.5`입니다.
- 엔드 중심 블록은 설정 파일이나 인게임 명령으로 변경할 수 있습니다.

세 차원의 보더 크기는 항상 동일하게 동기화됩니다. 커스텀 차원은 1차 지원 범위에 포함되지 않습니다.

## 7. 명령어 목록

| 명령어 | 권한 | 설명 |
|---|---|---|
| `/advboard set <x> <y> <z>` | 일반 플레이어 | 오버월드 중심과 진행 소유자를 설정합니다. |
| `/advboard status` | 일반 플레이어 | 진행도, 설정값, 중심과 차원별 실제 보더 크기를 표시합니다. |
| `/advboard recalc` | 일반 플레이어 | 소유자의 발전과제를 전부 다시 검사하고 보더를 동기화합니다. |
| `/advboard reload` | 일반 플레이어 | 디스크의 설정 파일을 다시 불러옵니다. |
| `/advboard config` | 일반 플레이어 | 현재 인게임 설정을 표시합니다. |
| `/advboard config show` | 일반 플레이어 | `/advboard config`와 동일합니다. |
| `/advboard config initialDiameter <blocks>` | 일반 플레이어 | 초기 보더 크기를 변경합니다. |
| `/advboard config growthPerAdvancement <blocks>` | 일반 플레이어 | 발전과제당 증가량을 변경합니다. |
| `/advboard config endCenterBlock <x> <z>` | 일반 플레이어 | 엔드 중심 블록을 변경합니다. |

### `/advboard set`

```mcfunction
/advboard set ~ ~ ~
```

- 오버월드에서 플레이어가 직접 실행해야 합니다.
- 일반 권한으로 사용할 수 있습니다.
- 실행자를 진행 소유자로 지정합니다.
- 같은 월드에서 다시 실행하면 중심과 소유자가 새 값으로 변경됩니다.
- 다시 실행해도 발전과제 진행도는 초기화되지 않습니다.

### `/advboard status`

```mcfunction
/advboard status
```

각 정보가 한 줄씩 표시됩니다.

- 활성 여부
- 진행 소유자 UUID
- 완료 발전과제 개수
- 초기 크기와 과제당 증가량
- 목표 크기와 마지막 적용 크기
- 오버월드·네더·엔드 중심
- 각 차원의 실제 보더 크기
- 마지막 적용 오류

### `/advboard recalc`

```mcfunction
/advboard recalc
```

발전과제 상태와 보더 크기가 맞지 않을 때 사용합니다. 진행 소유자가 접속 중이어야 합니다.

### `/advboard reload`

```mcfunction
/advboard reload
```

외부 편집기로 `config/advancementborder.json`을 수정한 뒤 실행합니다. 설정값이 유효할 때만 새 설정을 불러옵니다.

## 8. 인게임 설정 변경

인게임 설정 명령은 일반 플레이어 권한으로 사용할 수 있습니다. 변경값은 즉시 `config/advancementborder.json`에 저장됩니다.

### 현재 설정 보기

```mcfunction
/advboard config
```

또는:

```mcfunction
/advboard config show
```

### 초기 보더 크기 변경

```mcfunction
/advboard config initialDiameter 5
```

기본 시작 크기를 5블록으로 변경합니다. 증가량이 2라면 `5 → 7 → 9 → 11`로 성장합니다.

### 발전과제당 증가량 변경

```mcfunction
/advboard config growthPerAdvancement 1
```

증가량을 1블록으로 변경합니다. 초기 크기가 1이면 `1 → 2 → 3 → 4`로 성장합니다.

### 엔드 중심 변경

```mcfunction
/advboard config endCenterBlock 100 0
```

엔드 보더의 중심 블록 X/Z를 변경합니다.

### 변경값 적용 시점

- 월드보더가 아직 설정되지 않았다면 설정 파일에만 저장되고 이후 `/advboard set`에서 사용됩니다.
- 월드보더가 활성 상태이고 소유자가 접속 중이면 즉시 재계산됩니다.
- 소유자가 오프라인이면 설정을 저장한 뒤 다음 접속 시 재계산됩니다.
- 크기가 줄어드는 변경이라면 안전 이동 검사를 먼저 수행합니다.
- 안전한 이동 위치를 찾지 못하면 설정은 저장되지만 현재 보더 적용은 중단되며 `/advboard status`에 오류가 표시됩니다.

## 9. 설정 파일 편집

설정 파일 경로:

```text
config/advancementborder.json
```

기본 설정:

```json
{
  "schemaVersion": 1,
  "initialDiameter": 1,
  "growthPerAdvancement": 2,
  "endCenterBlock": {
    "x": 100,
    "z": 0
  }
}
```

| 필드 | 형식 | 기본값 | 설명 |
|---|---:|---:|---|
| `schemaVersion` | 정수 | `1` | 설정 파일 형식 버전입니다. 임의로 변경하지 마세요. |
| `initialDiameter` | 1 이상의 정수 | `1` | 발전과제 0개일 때의 보더 한 변 길이입니다. |
| `growthPerAdvancement` | 1 이상의 정수 | `2` | 유효 발전과제 1개당 추가되는 전체 보더 길이입니다. |
| `endCenterBlock.x` | 정수 | `100` | 엔드 중심 블록 X입니다. |
| `endCenterBlock.z` | 정수 | `0` | 엔드 중심 블록 Z입니다. |

파일을 직접 수정한 뒤에는 다음 명령을 실행하세요.

```mcfunction
/advboard reload
```

JSON 문법이 잘못되거나 최소값보다 작은 값이 있으면 기존 설정을 유지하고 한글 오류 메시지를 출력합니다.

## 10. 안전 이동 동작

발전과제 취소나 설정 변경으로 보더가 줄어들 때 소유자가 새 보더 밖에 있으면 안전한 위치로 이동합니다.

오버월드와 엔드에서는 다음 기준을 사용합니다.

- 목표 보더 안쪽의 지표면
- 발밑에 단단한 바닥이 있는 위치
- 발과 머리 공간에 충돌 블록이 없는 위치
- 물이나 용암이 없는 위치
- 나뭇잎, 마그마, 불, 모닥불, 선인장, 가루눈 등 위험 블록이 아닌 위치

네더에서는 기반암 천장 위로 이동하지 않도록 Y 120 이하에서 가장 높은 안전 위치를 찾습니다.

안전한 지표면을 찾지 못하면 플레이어를 동굴이나 지하로 강제 이동시키지 않습니다. 대신 보더 변경을 중단하고 기존 크기를 유지하며 `/advboard status`의 `마지막 오류` 항목에 이유를 표시합니다.

## 11. 권장 플레이 진행 순서

1. 월드를 생성하고 오버월드 시작 위치를 고릅니다.
2. `/advboard set ~ ~ ~`을 실행합니다.
3. `/advboard status`에서 초기 크기와 세 차원 동기화 상태를 확인합니다.
4. 일반 생존 플레이로 비루트 발전과제를 완료합니다.
5. 필요할 때 `/advboard status`로 완료 수와 보더 크기를 확인합니다.
6. 성장 속도를 변경하려면 `/advboard config` 명령을 사용합니다.
7. 설정이나 발전과제 상태가 맞지 않는다고 판단되면 `/advboard recalc`를 실행합니다.
8. 게임을 정상적으로 저장 종료합니다. 진행 소유자, 중심, 완료 수와 적용 크기는 월드에 저장됩니다.

## 12. 문제 해결

### `/advboard set`이 빨간색으로 표시됩니다

`set`에는 X/Y/Z 좌표가 모두 필요합니다.

```mcfunction
/advboard set ~ ~ ~
```

`/advboard set`까지만 입력하면 미완성 명령이므로 빨간색으로 보일 수 있습니다.

### `/advboard config`, `recalc`, `reload`가 보이지 않습니다

모든 AdvancementBorder 명령은 일반 권한으로 표시되어야 합니다. 최신 모드 JAR이 설치되어 있는지 확인하고, 이전 버전에서 교체했다면 게임이나 서버를 완전히 재시작하세요.

### `/advboard set` 실행에 실패합니다

- 플레이어가 직접 실행했는지 확인하세요.
- 오버월드에서 실행했는지 확인하세요.
- 좌표를 세 개 모두 입력했는지 확인하세요.

네더나 엔드에서 실행하면 오버월드 중심이 모호해지므로 거부됩니다.

### 발전과제를 완료했는데 보더가 커지지 않습니다

다음을 확인하세요.

- 루트 발전과제가 아닌지 확인합니다.
- 발전과제 화면에 표시 가능한 과제인지 확인합니다.
- 모든 조건을 완료했는지 확인합니다.
- `/advboard set`을 실행한 소유자의 발전과제인지 확인합니다.
- `/advboard status`에서 완료 개수가 늘었는지 확인합니다.

필요하면 다음 명령을 실행하세요.

```mcfunction
/advboard recalc
```

### 발전과제를 달성해도 채팅 알림이 없습니다

정상 동작입니다. 플레이 중 채팅이 도배되지 않도록 자동 보더 크기 로그는 출력하지 않습니다.

```mcfunction
/advboard status
```

위 명령으로 현재 크기를 확인하세요.

### 설정 파일을 수정했는데 반영되지 않습니다

```mcfunction
/advboard reload
```

그래도 반영되지 않으면 JSON 쉼표·괄호와 각 값의 최소 범위를 확인하세요. `initialDiameter`와 `growthPerAdvancement`는 1 이상이어야 합니다.

### 보더가 줄어들지 않습니다

소유자가 새 보더 밖에 있고 안전한 지표면을 찾지 못하면 플레이어 보호를 위해 축소가 취소될 수 있습니다. `/advboard status`의 마지막 오류를 확인하고 안전한 장소로 이동한 뒤 `/advboard recalc`를 실행하세요.

### 재접속 후 크기가 다릅니다

월드가 시작될 때 마지막 보더 상태를 복원하고, 진행 소유자가 접속하면 실제 발전과제 상태로 다시 계산합니다. 설정 파일을 변경했거나 발전과제 데이터팩이 달라졌다면 재계산 결과가 달라질 수 있습니다.

## 13. 자주 묻는 질문

### 치트 없이 플레이할 수 있나요?

가능합니다. 모든 AdvancementBorder 명령은 일반 권한으로 사용할 수 있습니다.

```mcfunction
/advboard set ~ ~ ~
/advboard status
```

월드 생성 시 `Allow Commands`를 켜지 않아도 중심 설정, 상태 확인, 성장 규칙 변경, 강제 재계산과 설정 재로딩을 모두 사용할 수 있습니다.

### 시작 크기를 1이 아닌 값으로 사용할 수 있나요?

가능합니다.

```mcfunction
/advboard config initialDiameter 5
```

또는 설정 파일의 `initialDiameter`를 변경하세요.

### 한 발전과제당 1블록씩 늘릴 수 있나요?

가능합니다.

```mcfunction
/advboard config growthPerAdvancement 1
```

초기 크기가 1이라면 `1 → 2 → 3 → 4`로 증가합니다.

### 여러 플레이어의 발전과제를 합산하나요?

아닙니다. `/advboard set`을 실행한 한 명의 소유자만 추적합니다. 다른 플레이어가 발전과제를 완료해도 보더에는 반영되지 않습니다.

### 다른 모드의 발전과제도 포함되나요?

발전과제 화면에 표시 가능한 비루트 과제라면 포함됩니다. 모드팩에 발전과제가 많으면 보더가 더 빠르게 커질 수 있습니다.

### `/advancement revoke`로 과제를 제거하면 어떻게 되나요?

현재 유효 완료 수에 맞춰 보더가 줄어듭니다. 플레이어가 새 보더 밖에 있으면 지표면 안전 위치를 찾은 뒤 이동합니다.

### 월드마다 중심을 다르게 정할 수 있나요?

가능합니다. 중심과 진행 소유자는 각 월드 세이브에 별도로 저장됩니다. 다만 `config/advancementborder.json`의 성장 규칙은 같은 게임 인스턴스에서 공통으로 사용됩니다.

---

빠른 명령 참고:

```mcfunction
# 현재 위치를 시작점으로 설정
/advboard set ~ ~ ~

# 상태 확인
/advboard status

# 설정 확인
/advboard config

# 1블록씩 성장하도록 변경
/advboard config growthPerAdvancement 1

# 진행도 강제 재계산
/advboard recalc
```
